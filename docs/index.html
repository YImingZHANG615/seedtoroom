种子到房间配置表

1. 项目简介

本项目演示了一个“医院智能规划系统”的基本思路，可根据“多维度种子信息”自动推算医院各类功能科室、病房、配套空间的理想配置，并在预算不足时按照优先级进行“舍弃或缩减”，输出一个相对可行的方案。主要特色：

  1. 多维度数据驱动：种子涵盖人口、疾病、文化宗教、婚育、基础设施、社会环境、经济和气候 8 大方面，信息丰富；
  2. 全面功能科室：不仅考虑床位数，也纳入门诊、急诊、检验科、手术室、妇产科、传染病科、宗教礼拜室、太平间、洗衣房等；
  3. 预算裁剪：若项目总造价超过 X:budget，就会从低优先级到高优先级依次减少房间数量，保证关键科室最少保留；
  4. 可扩展：可灵活扩展疾病种类、建筑防灾需求、精细化施工成本或更复杂的模型算法（如多目标规划）。

2. 种子结构与解析

本项目使用了一个多维度种子结构，由以下 8 个大类（维度） 组成，每个维度以单个大写字母表示，维度之间以竖线 | 分隔，具体如下：

  1. P - 人口 (Population)
  2. H - 健康威胁 (Health Threat)
  3. C - 文化宗教 (Culture & Religion)
  4. M - 婚育 (Marriage & Fertility)
  5. I - 基础设施 (Infrastructure)
  6. S - 社会环境与冲突 (Social Environment & Security)
  7. X - 经济 (Economy)
  8. G - 气候环境 (Geography & Climate)

同一维度内部的参数以逗号 , 分隔。下面对这 8 个维度的参数做详细说明，并给出示例。

2.1 人口 (P)

  • 用途：描述区域总人口、年龄结构、性别比例等，用来初步估算医院总体规模及专科床位需求。
参数 数据类型 示例 说明/影响
pop val pop=300000 总人口数，直接决定医院规模及总床位需求。
age0_14 ratio age0_14=0.35 0–14岁占比，决定儿科、疫苗接种和小儿重症需求。
age15_64 ratio age15_64=0.55 15–64岁占比，主要涉及内/外科、急诊等常见疾病的需求。
age65_up ratio age65_up=0.1 ≥65岁占比，慢性病、康复及长期护理需求较高。
gender val gender=1.02 性别比（男:女），女性较多时对产科/妇科需求更大。

2.2 健康威胁 (H)

  • 用途：描述主要疾病及健康风险，是各专科和隔离病房规划的核心。本维度支持同时记录多种高发病，每病种有发病率和现有床位数等信息。

2.2.1 多病种写法

  • 示例格式：
    H:dis=I(inc=300,res=0.2),Ma(inc=150,res=0.1),A(inc=80,res=0.05),...,risk=HP
    • inc：发病率；
    • res：已有专科床位；
    • risk：该区域健康风险，如高污染(HP)、吸烟(SM) 等，对发病加成。

2.2.2 常见疾病代码清单

2.3 文化宗教 (C)

  • 用途：反映当地宗教构成、性别隔离、传统医疗依赖及节假日风险，对就医行为与病房布局有直接影响。
参数 数据类型 示例 说明/影响
rel code rel=M0.7C0.2T0.1 宗教构成 (穆斯林、基督教、传统)，影响性别分区等
sexsep level sexsep=H 性别隔离等级 (L/M/H)，H 表示必须分设男女病区
trad ratio trad=0.9 传统医疗依赖度 (0~1)，高→住院日长
hol code hol=MHR 节假日风险(LHR/MHR/HHR)，高→需应急床位

2.4 婚育 (M)

  • 用途：描述生育率、婚育年龄、妇幼健康状况，决定产科、新生儿床位规划。
参数 数据类型 示例 说明/影响
fert val fert=3.2 生育率，越高→产科、新生儿科需求越大
mar val mar=19 初婚年龄，过低→早婚早育，对产科需求峰值影响大
health val health=18 妇幼健康指数，低→产科并发症多，需要加强重症监护

2.5 基础设施 (I)

  • 用途：反映医疗设施覆盖率、饮水卫生、交通条件，决定医院功能是否要“大而全”。
参数 数据类型 示例 说明/影响
fac val fac=0.5 基础医疗覆盖率，低→本院必须扩大功能
water ratio water=0.4 饮水卫生差→水源性疾病多，需强化传染病科或净水系统
infra ratio infra=0.2 交通不便→转诊难，本院需手术等综合功能

2.6 社会环境与冲突 (S)

  • 用途：描述社会稳定、冲突与难民情况，对急诊、外伤科、安全设计影响显著。
参数 数据类型 示例 说明/影响
conflict level conflict=L 冲突程度 (L/M/H)，高→急诊/外伤床位需求多，安全防护要求高
ref ratio ref=0.02 难民或流动人口比例，高→应急床位、挂号窗口、候诊区需增
vio val vio=0.01 暴力事件率，高→创伤与外科急诊需求
trust ratio trust=0.6 社会信任度，低→医患纠纷风险高，需要安保或调解室等

2.7 经济 (X)

  • 用途：反映经济状况与可用预算，决定建设规模、材料与设备采购水平。
参数 数据类型 示例 说明/影响
gdp val gdp=800 人均收入，低→建设资金不足，设备档次受限
pov ratio pov=0.4 贫困率，越高→患者支付能力弱，需更多公益支持
emp code emp=A0.7I0.2S0.1 就业结构：70%农业→农具外伤、寄生虫病较多
budget val budget=5 建设总预算 (百万美元)，超出则需削减或合并功能

2.8 气候环境 (G)

  • 用途：描述气候与自然灾害风险，对医院建筑设计、防护措施影响重大。
参数 数据类型 示例 说明/影响
temp val temp=32 年均气温 (°C)，高→需加强通风、空调等降温措施
rain val rain=400 年降水量 (mm)，大→需注重排水、防洪设计
disrisk code disrisk=F 灾害风险：F(洪涝)、D(干旱)、E(地震)、W(风暴)，影响结构加固
mat code mat=BC 建材 (BC砖混 / WM木泥 / RC钢混等)，影响造价与耐久性

2.9 示例种子 

P:pop=500000,age0_14=0.28,age15_64=0.62,age65_up=0.1,gender=1.02|
H:dis=I(inc=300,res=0.2),Ma(inc=150,res=0.1),A(inc=80,res=0.05),Tb(inc=100,res=0.06),risk=HP|
C:rel=M0.7C0.2T0.1,sexsep=H,trad=0.9,hol=MHR|
M:fert=3.2,mar=19,health=18|
I:fac=0.5,water=0.4,infra=0.2|
S:conflict=L,ref=0.02,vio=0.01,trust=0.6|
X:gdp=800,pov=0.4,emp=A0.7I0.2S0.1,budget=5|
G:temp=32,rain=400,disrisk=F,mat=BC
  • 人口：50 万，0–14 岁占 28%，15–64 岁占 62%，65 岁以上占 10%，性别比 1.02；
  • 健康威胁：传染病 300/疟疾 150/艾滋 80/结核 100，均有一定床位短缺，risk=HP 表示高污染；
  • 文化宗教：70% 穆斯林，性别隔离要求高，传统医疗依赖度 0.9，节假日风险中等；
  • 婚育：生育率 3.2、初婚年龄 19、妇幼健康指数 18；
  • 基础设施：覆盖率 0.5，饮水 0.4，交通 0.2；
  • 社会环境：冲突低、难民 2%、暴力事件 0.01、信任度 0.6；
  • 经济：人均 800$, 贫困率 0.4，主要农业人口，就业 70% 农业，预算 5 百万美元；
  • 气候：年均 32℃，降水 400mm，洪涝风险 F，建材 BC(砖混)。

3. 公式与逻辑详解

在代码中，我们会用到一系列数学公式/规则，让每个参数都对医院科室和功能房间产生影响。这里给出常见的示例公式，便于理解：

3.1 基准床位

BaseBeds  =  (pop1000)×k\text{BaseBeds} \;=\; \left(\frac{\text{pop}}{1000}\right) \times kBaseBeds=(1000pop​)×k

  • pop：总人口；
  • kkk：每千人基础床位数，可取 2~3 作为示例。

3.2 按年龄段分配

PediatricBeds=BaseBeds×(age0_14)×α1,AdultBeds=BaseBeds×(age15_64)×α2,GeriatricBeds=BaseBeds×(age65_up)×α3.\begin{aligned} & \text{PediatricBeds} = \text{BaseBeds} \times (\text{age0\_14}) \times \alpha_1,\\ & \text{AdultBeds} = \text{BaseBeds} \times (\text{age15\_64}) \times \alpha_2,\\ & \text{GeriatricBeds} = \text{BaseBeds} \times (\text{age65\_up}) \times \alpha_3. \end{aligned}​PediatricBeds=BaseBeds×(age0_14)×α1​,AdultBeds=BaseBeds×(age15_64)×α2​,GeriatricBeds=BaseBeds×(age65_up)×α3​.​

  • α1,α2,α3\alpha_1, \alpha_2, \alpha_3α1​,α2​,α3​：修正系数；若 trad 较高(患者就医晚、住院时长长)→可增加这些系数。

3.3 妇产科床位

AnnualBirths  =  pop×(fert1000)\text{AnnualBirths} \;=\; \text{pop} \times \left(\frac{\text{fert}}{1000}\right)AnnualBirths=pop×(1000fert​)

  • 例如 fert=2 → 每千人 2 例出生 / 年。
  • 若 mar<18 或 health<50，可额外加成→产科并发症多。

MaternityBeds  =  AnnualBirths×β,DeliveryRooms  =  max⁡(1,  ⌈AnnualBirths/CapacityPerRoom⌉)\text{MaternityBeds} \;=\; \text{AnnualBirths}\times \beta,\quad \text{DeliveryRooms} \;=\; \max(1,\; \lceil \text{AnnualBirths}/\text{CapacityPerRoom} \rceil )MaternityBeds=AnnualBirths×β,DeliveryRooms=max(1,⌈AnnualBirths/CapacityPerRoom⌉)

3.4 传染病 / 特殊病种床位

从 H:dis 中逐个病种读取 inc(发病率)、res(已有床位)；
DiseaseExtra(d)=max⁡(0,  γ×(人口基数×inc)−(人口基数×res))\text{DiseaseExtra}(d) = \max\Big(0,\;\gamma \times (\text{人口基数}\times \text{inc}) - (\text{人口基数}\times \text{res})\Big)DiseaseExtra(d)=max(0,γ×(人口基数×inc)−(人口基数×res))

  • 若 risk=HP(高污染) 或 infra<0.3(交通不便) → 可对 γ\gammaγ 做额外加成。

3.5 冲突 / 难民 / 暴力事件 => 应急床位

[
\text{EmergencyBeds} = \delta_1 \times \mathbf{1}(\text{conflict}=H)

  • \delta_2 \times (\text{ref}\times \text{pop})
  • \delta_3 \times (\text{vio}\times \text{pop})
    ]
  • conflict=H 时，外伤科或急诊床位加成；
  • ref(难民比) 较高 → 预留应急病房；
  • vio(暴力事件率) 高 → 更多创伤床位或 ICU。

3.6 性别隔离 (sexsep)

若 sexsep=H，在部分科室翻倍或增加隔离空间：

Rooms_withSexSep=Rooms×(1+θ)\text{Rooms\_withSexSep} = \text{Rooms}\times (1 + \theta)Rooms_withSexSep=Rooms×(1+θ)

  • 例如 θ=0.2∼1.0\theta=0.2 \sim 1.0θ=0.2∼1.0，看需求设定。

3.7 预算约束

先算出理想状态下所有房间/科室数量与单间造价，然后与 budget(百万美元) 对比：

TotalCost=∑(RoomCounti×UnitCosti)\text{TotalCost} = \sum (\text{RoomCount}_i \times \text{UnitCost}_i)TotalCost=∑(RoomCounti​×UnitCosti​)

  • 若 TotalCost>budget×106\text{TotalCost} > \text{budget}\times 10^6TotalCost>budget×106，按照优先级从低到高依次削减。
  • 削减策略：对于低优先级(如 3)房间，每次减少 1 间，直到资金满足或该项变为 0；然后再削减优先级=2；对优先级=1（公共卫生关键），至少保留 1 间不被砍光。

4. 使用方法与拓展

  1. 环境：Python 3.7+，无第三方库依赖（仅用到 math、re 标准库）。
  2. 运行：
    • 执行 python hospital_planning.py；
    • 可看到控制台输出理想房间配置及最终预算内配置。
  3. 自定义：
    • 修改种子：可在 main() 中的 seed_example 替换为自己地区的数据；
    • 完善公式：在 compute_rooms_with_all_params() 中，根据更详细的医疗规范或实际数据，修改加成/系数；
    • 细分优先级：priority 可以设为 1~5 或更多级别；若要更复杂的缩减策略，可自定义 apply_budget_cut()；
    • 结合外部数据：可与 GIS、统计部门数据、WHO 指南等衔接，建立更加完善的规划系统。

5. 最后声明

  • 示例性质：本文所用公式、系数、优先级、单位造价等均为示范之用，不是权威标准。
  • 专业性：实际落地需要参考当地法规、公共卫生数据、医疗建筑规范，并与专业分析结合进行细化。
